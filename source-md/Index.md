Milvus 等向量数据库通过构建不同的索引结构来高效地实现 ANN 搜索。

### ① IVF_FLAT (基于聚类的索引)

|**特征**|**IVF (Inverted File Index)**|
|---|---|
|**全称**|Inverted File Index, Flat (倒排文件索引)|
|**工作原理**|是一种**基于聚类**的索引。它在构建时将所有向量通过 K-Means 等算法分成 $N_{list}$ 个**簇**。查询时，它只会搜索距离查询向量最近的 $nprobe$ 个簇（而不是全部簇），并在这些簇内进行精确比较。|
|**准确度调优**|增大搜索参数 `nprobe`（查询的簇数量），搜索范围扩大，**召回率提高**，但速度变慢。|
|**优势**|索引构建速度快，内存消耗相对较低。|
|**劣势**|搜索速度受限于 `nprobe`，且在高维数据或极高召回率要求下性能不如 HNSW。|

### ② HNSW (基于图的索引)

| **特征**    | **HNSW (Hierarchical Navigable Small World)**                                         |
| --------- | ------------------------------------------------------------------------------------- |
| **全称**    | Hierarchical Navigable Small World (分层可导航小世界)                                         |
| **工作原理**  | 是一种**基于图结构**的索引。它将所有向量组织成一个**多层图**：顶层稀疏，用于快速导航；底层密集，用于精确搜索。查询时，它在图上进行贪婪搜索，可以快速找到最近邻域。 |
| **准确度调优** | 增大搜索参数 `ef`（搜索范围），搜索路径更广，**召回率提高**，但内存和计算消耗增加。                                        |
| **优势**    | 在搜索速度和召回率方面表现**极其优异**，尤其适用于高吞吐量、低延迟的实时查询场景。                                           |
| **劣势**    | 索引构建速度较慢，索引文件和内存消耗通常比 IVF 索引高。                                                        |
