0.基础
0.1 和SFT的区别？
- SFT 的局限性：SFT 依赖数据标注的质量和覆盖性，泛化性有限，需覆盖场景内所有数据才能达到效果；存在机械模仿和创造性不足的问题，难以处理多目标、多答案的情况；无法进行拒绝，模型只能知道下一个 TOKEN 是什么，不知道不应该是什么。
- RLHF 的优势：RLHF 通过奖励模型将人类偏好融入其中，能覆盖不同 query，与人类偏好对齐；可对所有正确答案打高分，随机生成答案，处理多目标、多答案问题；能拒绝有害回答，解决实际生产场景中的 Badcase。

0.2 什么时候用强化学习
1.1 “评价容易，但生成很难”的场景（Verification is easier than Generation）
这是强化学习最经典的应用场景。例如数学推理（Math）和代码生成（Code）。
- 原因：让人类写出完美的复杂数学推导过程极度昂贵；但写一个程序来验证最终答案是否正确、或者验证代码能否编译通过并跑通测试用例，成本几乎为零。
- 做法：用规则作为Reward，让模型自己生成大量解法（探索），对结果正确的解法给予高奖励（RLOO / PPO），这就是目前OpenAI o1等推理模型的核心逻辑。
1.2 需要对齐人类复杂偏好，答案无绝对标准（Alignment）
例如要求回复“幽默、礼貌、无害”。
- 原因：这类问题没有唯一的“标准答案”，SFT很难教模型什么叫“幽默”。但是人类可以轻易在两个回复中选出“哪一个更幽默”。
- 做法：基于比较数据训练一个奖励模型（Reward Model），然后通过RLHF引导模型向高分区间靠拢。
1.3 需要严厉惩罚特定“负面行为”（幻觉与安全性）
- 原因：SFT只能告诉模型“该说什么”，很难告诉模型“绝对不能说什么”。如果不慎在SFT数据里混入了一条有害数据，模型就会学进去。
- 做法：通过RL，给模型的“拒绝回答”或“安全回复”正反馈，给“有害输出”极大的负反馈惩罚，能更有效地抑制幻觉和保证模型安全性。
1.4 追求“超越人类上限”的超级智能（Self-Play / Super-Alignment）
- 原因：当模型能力达到一定高度，人类专家已经写不出比模型更好的SFT数据了（例如极高难度的算法题、复杂的战略博弈AlphaGo）。
- 做法：让模型左右互搏（Self-Play），通过胜负作为Reward进行强化学习，实现能力的自我进化。

0.3 马尔可夫决策过程（MDP）中的基本元素
1. 智能体（Agent）：学习和决策的核心主体。它通过与环境的交互来学习最优的行为策略，以实现特定的目标。
2. 状态空间（States, S）：系统所有可能状态的集合。
3. 动作空间（Actions, A）：在每个状态下可以采取的所有可能动作的集合。
4. 状态转移概率（Transition Probability, P）：给定当前状态 s 和动作 a，转移到下一状态 s′ 的概率，记为 P(s′∣s,a)。
5. 奖励函数（Reward Function, R/r）：在状态 s 下采取动作 a 后，转移到状态 s′ 所获得的即时奖励，记为 R(s,a,s′)。
6. 累积奖励（Cumulative reward，G）：在t时刻采取行动a在未来能获得的奖励，不考虑衰减的奖励$$G_t= \sum_{k=0}^{n} R_{t+k}$$，考虑衰减的奖励$$G_t= \sum_{k=0}^{n} \gamma^kR_{t+k}$$
7. 折扣因子（Discount Factor, γ）：用于控制未来奖励的重要性，取值范围为 [0,1]。折扣因子越接近1，表示对未来奖励的重视程度越高
8. 策略（Policy，Π）：在状态s采取动作a的概率，即：$$\pi(a|s) = p(A_t = a| S_t = s)$$。现实任务一般都是随机性策略，即在一个状态下按照概率采取不同动作，而非只能采取唯一动作的确定性策略。

0.4 MDP的过程
一、 核心前提：马尔可夫性质 (Markov Property)
在构建整个系统之前，我们必须先确立一个宇宙定理，即马尔可夫性质。
一言以蔽之：“未来只与现在有关，与过去无关。”
在数学上，这意味着系统在 $$t+1$$ 时刻的状态，仅仅取决于它在 $$t$$ 时刻的状态和采取的动作，而不需要追溯 t-1、t-2 等历史时刻的信息。当前状态已经包含了预测未来所需的全部历史信息。这是极大简化计算复杂度的核心前提。
二、 舞台与演员：状态、动作与智能体
在一个强化学习任务中，首先需要定义谁在做事，以及它能做什么和看到什么。
1. 智能体 (Agent)
- 定义：它是学习和决策的核心主体（即“大脑”）。
- 作用：Agent 不断地观察环境，做出决策，采取行动，并根据环境反馈的奖励来更新自己的知识，最终目的是找到一套能获得最大利益的做事准则。
2. 状态空间 (States, $$S$$)
- 定义：环境所有可能呈现的形态集合。
- 通俗理解：这是 Agent 能够“看”到的环境信息。比如下棋时的棋盘残局、自动驾驶时的摄像头画面和车速、机器人所处的坐标位置。当前时刻的状态通常记为 $s$。
3. 动作空间 (Actions, $$A$$)
- 定义：在任何一个特定状态下，Agent 可以采取的所有合法行为的集合。
- 通俗理解：这是 Agent 能够“做”的事情。比如马里奥游戏里的“上下左右跳”、自动驾驶里的“方向盘角度和油门力度”。当前时刻采取的动作通常记为 $a$。
三、 世界的运行法则：转移概率与奖励
当 Agent 在状态 $$s$$ 采取了动作 $$a$$ 之后，环境会发生变化，并给出反馈。这种环境的内在机制由两个函数决定：
4. 状态转移概率 (Transition Probability, $$P$$)
- 定义：环境的动态演变规律。给定当前状态 $$s$$ 和动作 a，系统转移到下一个特定状态 $$s'$$ 的概率，记作
 P(s' | s, a)。
- 通俗理解：现实世界充满不确定性。比如你在一片结冰的湖面上决定“向前走一步”（动作 a），你可能成功前进了一步（状态 s'），但也可能会滑倒在原地甚至后退（其他状态）。P就是描述这种不确定性的概率分布。
5. 奖励函数 (Reward Function, $$R$$)
- 定义：环境对 Agent 特定行为的即时反馈（标量信号）。在状态 $$s$$ 下采取动作 $$a$$ 后，转移到状态 $$s'$$ 时，环境给予的即时奖励记为 R(s, a, s')（通常简写为 $$R(s, a)$$ 即期望奖励）。
- 通俗理解：这是强化学习中最核心的引导信号。做对了给正反馈（如得分、抵达终点），做错了给负反馈（如扣血、撞车）。它直接定义了任务的目标是什么。
四、 目标的度量：目光长远与累积收益
有了即时奖励后，Agent 需要计算长远收益，因为很多时候“短期的牺牲是为了长远的利益”。
6. 折扣因子 (Discount Factor, $$\gamma$$)
- 定义：一个范围在 $$[0, 1]$$ 之间的超参数，用于衡量“未来的奖励”在“当前”的价值贬值程度。
- 通俗理解：类似于金融里的“通货膨胀”或“折现率”。
  - 当 $$\gamma = 0$$ 时，Agent 变得极其短视，只在乎眼前的即时奖励（贪心算法）。
  - 当 $$\gamma$$ 接近 1（如 0.99）时，Agent 变得非常有远见，愿意为了未来的大奖而放弃眼前的蝇头小利。
7. 累积奖励 / 回报 (Cumulative Reward / Return, $$G_t$$)
- 定义：从当前时刻 $$t$$ 开始，直到一回合（Episode）结束，所有带衰减的未来奖励的总和。
  $$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \dots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}$$
- 核心意义：Agent 真正在乎的不是某一步的即时奖励 $R$，而是这个代表终极目标的一生总收益 $G_t$。强化学习的终极目标就是最大化累积奖励 $$G_t$$ 的期望值。

---

五、 最终的输出：策略 (Policy)

所有的观察、行动、概率和计算，最终都是为了凝练出一条指导未来的法则。
8. 策略 (Policy, $$\pi$$)
- 定义：Agent 的行为准则，即在给定状态 $$s$$ 下，采取各类动作 $$a$$ 的概率分布：$$\pi(a|s) = P(A_t = a | S_t = s)$$
- 通俗理解：它就是 Agent 训练出来的“直觉”或“说明书”。当你把当前的状态 $$s$$ 输入给这本说明书时，它会告诉你下一步应该干什么。
- 确定性与随机性：
  - 确定性策略：在状态 $$s$$ 下，永远输出唯一确定的动作 $a$（比如红灯绝对要停）。
  - 随机性策略（现实中最常用）：输出的是各个动作的概率（比如有 80% 的概率向左走，20% 的概率向右走）。这不仅使动作更加平滑，还能保留 Agent 探索未知环境的能力。
总结
至此，一个完整的马尔可夫决策过程 (MDP) 就构建完毕了，它由一个五元组 $$\langle S, A, P, R, \gamma \rangle$$ 构成。
整个强化学习的过程可以浓缩为一句话：
Agent 在状态空间 $$S$$ 中，依据某种策略 $$\pi$$ 挑选动作 $$A$$；环境接收动作后，按照状态转移概率 $$P$$ 进入下一个状态，并吐出即时奖励 $$R$$；Agent 结合折扣因子 $$\gamma$$ 计算出累积奖励 $$G$$ 的好坏，并以此为根据不断修改自己的策略 $$\pi$$，直到找到那个能吃到最多奖励的最优策略 $$\pi^*$$ 为止。

0.5 强化学习的分类方式
1. 懂不懂环境？（是否对环境建模）
- 无模型 (Model-Free)：不学习环境规律（如转移概率），直接在环境里“盲人摸象”试错，简单直接，大模型主流（代表：DQN, PPO）。
- 有模型 (Model-Based)：先建构虚拟环境进行“沙盘推演”再做决定，极度节约数据，但环境模型一旦不准结果极差（代表：AlphaZero）。
2. 脑子里存什么？（优化目标/架构）
- 基于价值 (Value-Based)：不直接输出动作，只给所有动作“打分”选最高，仅限离散动作空间（代表：Q-Learning, DQN）。
- 基于策略 (Policy-Based)：直接学习策略网络，输出动作的“概率”，擅长处理连续动作空间（代表：REINFORCE）。
- Actor-Critic (演员-评论家)：Actor负责输出动作概率，Critic负责给动作打分，收敛稳方差小，工业界主力（代表：PPO, SAC）。
3.用什么数据
-  同策略 (On-Policy)：收集数据和被优化的策略是同一个大脑。即“边做边学”，产生的数据用完必须扔掉。训练稳定，但极度费数据（样本效率低）。
-  异策略 (Off-Policy)：收集数据和被优化的策略不是同一个。有一个经验回放池（Replay Buffer），可以“吸取历史经验”。数据能反复用（样本效率极高），但训练容易不稳定。

怎么区分On-line和  Off-line？
- On-line
  - 定义： 算法在训练过程中持续地与环境进行交互，实时或在短时间内使用新收集的数据进行学习更新。
  - 特点：
    - 数据收集和模型训练是交织进行的。
    - 需要一个实时的环境模拟器或真实的物理环境。
    - 模型（策略或价值函数）会随着训练的进行而改变，并且数据是基于当前模型与环境的交互。
  - 典型场景： 机器人学习行走、玩游戏（游戏模拟器）、自动驾驶（在模拟或真实环境中）。
- Off-line
  - 定义： 算法仅使用一个预先收集好的、固定的数据集进行训练，在训练过程中不与环境进行任何新的交互。
  - 特点：
    - 数据收集阶段与模型训练阶段是完全分离的。
    - 不需要实时环境，只需要一个数据集。
    - 面临数据分布偏移 (Distribution Shift) 的挑战：训练数据分布可能与目标策略在环境中实际产生的数据分布不同，可能导致学习到的策略在实际环境中表现不佳。
  - 典型场景： 从历史用户交互日志、机器人操作记录、医疗数据等中学习策略，这些数据量通常很大且收集成本高，难以进行额外的在线探索。这是一个新兴且活跃的研究领域（常被称为 Batch RL 或 Offline RL）。

0.6 介绍Actor和Critic分别是什么，以及常用的critic模型
- Actor 负责与环境交互采集数据，并在 Critic 价值函数的指导下用策略梯度学习一个更好的策略（类似于策略优化）。
- Critic 通过 Actor 与环境交互收集的数据学习一个价值函数（类似于策略评估），用于判断状态和动作的好坏。
常用的Critic模型有
- 轨迹的总回报：需要整个轨迹计算完才能得到，缺乏即时反馈。此外，对所有时刻都采用同样的critic值，缺乏对时间维度的区分度。
- 动作$$a_t$$之后的回报：就是REINFORCE算法，需要完整的episode，episode生成和策略更新不可同时进行，训练效率较低。
- 加入基准线的REINFORCE，能减小方差
- 动作价值函数：直接学习Q网络
- 优势函数
- 时序差分误差

Actor-Critic
Actor-Critic 的本质是结合了策略梯度和价值函数的双网络架构，用价值网络的评估信号代替环境原始奖励来指导策略网络的更新，以此显著降低策略梯度的方差。

Actor-Critic 由两个神经网络协同工作，核心逻辑分三步。首先，**Actor网络（策略端）**负责根据当前状态直接计算并输出具体动作的概率分布，引发环境交互。接着，Critic网络（价值端）通过时序差分（TD）等算法拟合环境反馈的真实奖励，计算出当前状态的价值预测或优势函数（Advantage）。最后，Actor 不再使用高方差的单次轨迹总奖励来更新参数，而是直接套用 Critic 计算出的稳定基准值去反向传播计算策略梯度，从而既保留了连续动作空间的决策能力，又大幅拉升了训练的稳定性。

但是...
工程实战中最让人绝望的是它的双网非平稳目标（Moving Target）问题。Actor 和 Critic 在同时更新，Critic 在尝试评估一个每一轮都在变的 Actor，很容易导致两个网络带着彼此往悬崖里带，死活不收敛。为了省显存，很多开源源码会把 Actor 和 Critic 强行共享底层特征提取层，这在复杂任务中极易引发严重的梯度冲突：动作网络关注特征差异，价值网络关注特征共性，最后提取出的表征全毁。因此大厂调参时，一般会把 Critic 的学习率设得比 Actor 大一个数量级，强迫价值评估先收敛，再稳稳地指导动作更新。

面试官可能追问与答案

Q1: Actor-Critic 里常说的“优势函数（Advantage）”解决了什么数学痛点？
它通过“当前动作实际得分”减去“当前状态平均预期得分（基线）”，把绝对价值转换成了相对价值。
这直接解决了一个致命痛点：如果环境奖励全是正数，那些没被采样到的优质动作的被选概率会被动下降。用 Advantage 引入负值后，表现低于平均水平的动作才会真正被惩罚。

Q2: 既然 Actor 和 Critic 容易梯度冲突，工程上怎么把底层骨干网络拆开？
工程上的标准解法是特征解耦（Decoupled Architecture），让 Actor 和 Critic 各自独享完整的网络主干。
代价是直接增加了一倍的显存占用和推理延迟。如果非要共享主干，必须在 Critic 的分支处使用 stop_gradient（截断梯度）操作，防止它去干扰底层特征提取。

Q3: 为什么大模型 RLHF 用的 PPO 本质上也是 Actor-Critic，却能训练得这么稳？
因为 PPO 在原生 Actor-Critic 的基础上，给 Actor 的更新幅度上了强制枷锁（Clip操作）。
它严格限制了新老策略的概率比值（通常控制在 0.8 到 1.2 之间），一旦 Actor 步子迈得太大，梯度直接被截断，从根本上锁死了双网络互相带崩的恶性循环。

0.7 重要性采样是什么？ 重要性采样在RL中的作用？
重要性采样是什么？
核心问题
想算某个分布 p 下的期望，但从 p 采样很难，从 q 采样容易：
$$E_{x \sim p}[f(x)] = E_{x \sim q}\left[\frac{p(x)}{q(x)} \cdot f(x)\right]$$
- 左边：想算的目标，从 p 采样
- 右边：实际做的，从 q 采样，但每个样本乘 p(x)/q(x) 修正
- p(x)/q(x) 就是 重要性权重，补偿"我用的不是 p 的数据"
💡 白话：旧策略选了动作 A，新策略选 A 的概率更高，说明这条数据对新策略更有参考价值，权重就调大；反之调小
在 RL 中的作用
没有重要性采样（纯 On-Policy）
用当前策略采样 → 更新一次 → 数据扔掉 → 重新采样...
每批数据只能用一次，极其浪费
用了重要性采样之后
原本目标函数必须用当前策略 π_θ 采样：
$$J(\theta) = E_{\tau \sim \pi_\theta}[R(\tau)]$$
改写成用旧策略 π_old 的数据：
$$J(\theta) = E_{\tau \sim \pi_{old}}\left[\frac{\pi_\theta(a|s)}{\pi_{old}(a|s)} \cdot R(\tau)\right]$$
- π_θ(a|s)：新策略选动作 a 的概率
- π_old(a|s)：旧策略选动作 a 的概率
- 两者之比：重要性权重，修正新旧策略差异
- R(τ)：这条轨迹的实际回报
✅ 结果：同一批数据可以复用多次，大幅提升样本效率
问题：权重会爆炸
新旧策略差异太大时，比值可能变成 10、100，训练直接崩掉
PPO 的解决方案：Clip 截断
令 r = π_θ / π_old，PPO 目标函数：
$$L = E\left[\min\left(r \cdot A, \ \text{clip}(r, 1-\varepsilon, 1+\varepsilon) \cdot A\right)\right]$$
- r：重要性权重
- clip(r, 0.8, 1.2)：把 r 强制限在 0.8~1.2，超出就截断
- 外面套 min：不管 A 正负，都阻止更新幅度过大 
  - A > 0（好动作）且 r 过大 → 取 clip 值，别更新太猛
  - A < 0（坏动作）且 r 过小 → 取 clip 值，同样限制
一句话总结
重要性采样让旧数据能代替新数据使用（乘修正权重）→ 解决了On-Policy数据浪费的问题
PPO的Clip 是在此基础上防止权重爆炸 → 保证训练稳定

0.8 GAE广义优势估计
更新策略时需要知道"这个动作比平均水平好多少"，也就是优势函数 A(s,a)。估计它有两种方式：
- MC（蒙特卡洛方法）：走完整轨迹用真实回报来算，无偏但方差大
- TD（时序差分算法）：走一步，用"当前奖励 + 下一个状态的预估价值"（rt+γV(st+1) ）来代替真实总收益。，方差小但 V 本身不准有偏差
两种都有缺陷，GAE 就是为了折中,GAE 的核心是多步 TD：不看一步，也不看完全部，而是看接下来的 N 步（比如 5 步），然后做加权平均。构造一个可调节的优势估计器，参数 λ∈[0,1]  控制偏差-方差权衡。
- 当 λ=0 ：只看1步，变成纯 TD（低方差高偏差）
- 当 λ=1 ：看完整局，变成纯 MC（无偏高方差）
- 当 λ=0.95 ：看 20 步左右，既减少方差（比 MC 稳），又减少偏差（比 TD 准）

0.9 RLHF是什么
RLHF，即基于人类反馈的强化学习，是一种结合强化学习算法与人类主观判断的训练技术。它通过引入人类的偏好和反馈来优化模型的行为和输出，使模型能够生成更符合人类期望的结果。
训练步骤为：
1. 预训练语言模型：首先需要一个预训练的语言模型，通过大量语料训练出基础模型。例如，ChatGPT的基础模型是GPT-3。
2. 训练奖励模型：收集人类标注的偏好数据，训练一个奖励模型（Reward Model, RM），该模型能够预测人类对不同输出的偏好分数。
3. 强化学习微调：利用奖励模型提供的奖励信号，通过强化学习算法（如PPO）对语言模型进行微调，优化模型的输出

0.10 RLHF的奖励函数相比于传统强化学习的奖励函数有什么特点
传统强化学习通过环境提供的奖励信号训练Agent，但复杂任务（如自然语言生成）往往难以设计自动化的奖励函数。RLHF 的核心创新在于用人类反馈替代传统奖励函数，通过人类对模型输出的评价，训练一个“奖励模型”（Reward Model），再用这个模型指导强化学习过程，最终让模型的输出更符合人类偏好。该奖励函数满足以下需求：
1. 能够解决人类 只能识别所需行为，但不一定能提供演示的任务
2. 允许 非专家用户 进行示教
3. 能扩展到大规模问题
4. 用户给出反馈的成本不高

0.11 RLVR是什么，和RLHF，传统的奖励函数对比，怎么用
RLVR 是一种主要用于大语言模型（LLM）后训练（Post-training）的强化学习范式。它的核心思想是：放弃训练一个基于人类偏好的神经网络奖励模型（Reward Model），转而使用程序化的、客观确定的规则（Verifier/验证器）来自动计算奖励。
典型的可验证场景包括：
- 数学问题：提取模型输出的最终答案，与标准答案（Ground Truth）做比对（如使用 SymPy 引擎判断代数等价性）。
- 代码生成：将模型生成的代码放入沙盒中运行，看是否通过了全部的单元测试（Unit Tests）。
- 格式遵循：强制模型按照 <think>...</think><answer>...</answer> 的格式输出，少一个标签就直接扣分。
[图片]
1. 大模型的“顿悟”与 RLVR 的关系 (DeepSeek-R1-Zero 现象)
  - 话术：“RLVR 最迷人的地方在于它可以引发涌现推理（Emergent Reasoning）。比如 DeepSeek-R1-Zero 完全没有使用 SFT（监督微调）数据，仅仅依靠 RLVR 纯强化学习，模型就自己学会了‘自我反思’、‘尝试后推翻重来’以及‘增加思考时间来换取准确率（Test-time Compute）。这是 RLHF 根本做不到的。”
2. RLVR 的局限性与现代训练管线 (Modern Training Pipeline)
  - 话术：“当然，RLVR 不是万能的。它只能教模型 ‘怎么做题 (How to solve)’，但无法教模型 ‘怎么好好说话 (How to behave)’。纯 RLVR 训练出来的模型往往语言可读性差，甚至会在一种语言的 prompt 下输出多种语言混合的思考过程。因此，现在的 SOTA Pipeline 通常是：先用 RLVR 提升极致的推理能力，然后再用少量高质量数据的 SFT 和 RLHF 来拉齐人类价值观和语言偏好。”
3. RLVR 到底是让模型变聪明了，还是只是提升了搜索效率？
  - 话术：“学术界最近对此有深入讨论。一些研究表明，RLVR 其实是将 Base 模型本身就具备但概率极低的‘正确推理路径’的概率大幅放大了。也就是说，它更像是一种高效的搜索空间压缩（Search Compression），它激发了模型的潜力边界，并提高了单次采样的通过率 (Pass@1)。”
它的核心优点是零人工标注成本，彻底杜绝奖励黑客（Reward Hacking），并能极大激发模型“自我反思、顿悟”等涌现推理能力（如DeepSeek-R1）；但缺点是适用场景窄（仅限有标准答案的客观题）、极易遭遇奖励稀疏（长逻辑最后一步算错全盘皆输），且容易导致输出语言混杂、过度思考与可读性崩塌，因此现代管线通常是先用RLVR拔高极致的推理智商，再用RLHF拉齐人类表达情商。

0.12 kl散度在RL中的作用
公式
KL 散度在 RLHF/RLVR 中的核心作用体现为对奖励函数的修正（Reward Shaping），通过数学公式将分布差异转化为实实在在的扣分项，构建起训练过程的“三重防护”。
其一，防止奖励黑客。全局锚定与防偏离机制。 最终奖励并非纯粹的外部评分 $R$，而是带有 KL 惩罚的综合奖励：
$$R_{total} = R - \beta \log\frac{\pi_\theta(y|x)}{\pi_{ref}(y|x)}$$
当当前策略 $$\pi_\theta$$ 为“刷分”而生成参考模型 $$\pi_{ref}$$ 认定极低概率的乱码（如 $$\pi_\theta=0.9$$ 但 $\pi_{ref}=0.001$）时，比值激增，经 $$\log$$ 与负系数 $$-\beta$$ 放大后产生巨大负惩罚，直接让偏离正常人类语言的行为无利可图。这强制策略始终“锚定”在初始 SFT 模型附近，既防止奖励黑客（Reward Hacking）（避免复读机式乱码），又防止灾难性遗忘，确保新技能学习不破坏原有语言能力与通用知识。

其二，防止灾难性遗忘。Trust Region 的“双重限速”协同。 在 PPO 框架下，Clip 机制与 KL 惩罚构建微观与宏观的双保险：Clip 限制 $$\frac{\pi_{new}}{\pi_{old}}$$，作为“短期微观刹车”防止单次 batch 更新过猛导致梯度爆炸；KL 惩罚限制 $$\frac{\pi_\theta}{\pi_{ref}}$$，作为“长期宏观缰绳”约束从训练起点至今的累积偏离。前者保证每一步走得稳，后者确保无论训练多久都不会走出安全区，共同实现稳健收敛。
其三，稳定训练过程。隐式熵正则与多样性维持。 预训练好的 $$\pi_{ref}$$ 具有健康的词汇分布（合理熵值）。若不加约束，模型为追求确定高分易将某词概率推至 1.0（熵坍塌）。而 KL 公式强制 $$\pi_\theta$$ 逼近 $$\pi_{ref}$$ 的分散分布——当模型试图集中概率时，与 $$\pi_{ref}$$ 的低概率差异会触发惩罚——从数学上强迫保留“多选词、多句式”的分布特性，在不显式添加熵奖励的情况下，天然平衡了探索（Exploration）与利用（Exploitation），防止模式崩溃（Mode Collapse）。

简言之，奖励函数负责“踩油门”指引方向，KL 散度负责“踩刹车”划定底线：它通过 log 比值惩罚机制杜绝乱码，依托双保险限速确保训练稳定，并以分布逼近的数学约束隐式维持生成多样性，是 RLHF 训练中不可或缺的安全绳与调节阀。
其核心作用有三点：一是防止奖励黑客（Reward Hacking），约束模型不要为了极限刷取高额奖励而生成非人类语言或无意义的乱码；二是防止灾难性遗忘，确保模型在学习新策略（如复杂的数学推理或对齐人类偏好）时，依然“锚定”在初始模型的基础之上，保留原有的语言流畅度、语法和通用知识；三是稳定训练过程（如TRPO/PPO的核心思想），通过限制每次策略更新的幅度（Trust Region），防止模型参数更新过猛导致训练崩溃，从而保证模型稳健收敛。简而言之，奖励函数负责“踩油门”指引方向，KL散度负责“踩刹车”保证模型不偏离正常轨道的底线。

0.13 RLHF的标准三阶段对齐程序
收集人类反馈：通过人类的示范和反馈来构建奖励模型。
强化学习：利用奖励模型在强化学习框架下优化模型的policy。
增强对齐：通过进一步的优化和迭代，使得模型更好地对齐人类的目标和价值观。

0.14  RLVR 的局限是什么
最新研究（NeurIPS 2025）揭示：RLVR 提升的是 pass@1，但会压缩探索空间。当采样次数 k 很大时（pass@k），基础模型往往能反超 RLVR 训练后的模型。
RLVR 并不是「教会了模型新能力」，而是「让模型更倾向于走已知的正确路径」。本质是提升采样效率，而非扩展能力边界。

1.PPO
1.1 RLHF-PPO 四个模型
- Actor Model：策略模型，这就是我们想要训练的目标语言模型。
- Reference Model：参考模型，它的作用是在RLHF阶段给语言模型增加一些“约束”，防止语言模型训歪。我们希望训练出来的Actor模型既能达到符合人类喜好的目的，又尽量让它和SFT模型不要差异太大。即希望两个模型的输出分布尽量相似，通过与Actor Model之间的KL散度控制。
- Critic Model：评估模型/价值模型，它的作用是期望回报，在RLHF中，我们不仅要训练模型生成符合人类喜好的内容的能力（Actor），也要提升模型对人类喜好量化判断的能力（Critic）。
- Reward Model：奖励模型，它的作用是计算即时收益。奖励模型可以是人为规定的，也可以用神经网络实现。
其中Actor和Critic Model是需要训练的，Reward和Reference Model是参数冻结的。Actor 和Reference model用同一个sft模型初始化，Reward 和Critic model用同一个奖励模型初始化。
1.2 NLP场景下的MDP建模
目的：给模型一个prompt，模型能生成符合人类喜好的回复。在时刻t产生一个token，t+1时刻的输入是t时刻的输入+t产生的token，如此自回归的生成。接下给出在NLP场景下强化学习的一些概念：
- 策略$$\pi(a_t|s_t):$$在状态$$s_t$$下生成token $$a_t$$的概率
- 动作 $$a_t$$：生成的token，动作空间是词表，
- 状态$$s_t$$：用户输入的prompt+t时刻前生成的所有token
- 奖励$$R_t$$：在状态$$s_t$$下生成token $$a_t$$的即时奖励。
- 价值$$V_t$$：状态价值函数，包含着即时奖励和未来期望奖励，$$V_t = E [G_t|s_t] = E( R_{t} + \gamma G_{t+1}|s_t)$$
- 训练目标：最大化期望回报，其中$$x, y$$分别表示用户查询和模型输出：
$$\mathop{\max}\limits_{\pi} E_{x \sim \mathcal{D}, y \sim \pi}[R(x, y)]$$
更常见的训练目标则是带有行为约束的强化学习，对策略更新幅度进行了限制，$$\pi_{ref}$$一般采用上一轮的$$\pi$$策略。
$$\mathop{\max}\limits_{\pi} E_{x \sim \mathcal{D}, y \sim \pi}[R(x, y) - \beta D_{KL}(\pi(y|x) || \pi_{ref}(y|x))]$$
NLP场景下状态转移是确定的，也就是在状态$$s_t$$下生成token $$a_t$$转移到的下一个状态$$s_{t+1}$$是固定的，即状态$$s_t$$拼接生成的token$$a_t$$。

1.4 RLHF-PPO的优缺点
RLHF-PPO 通过奖励模型将人类偏好转化为可优化信号，有效实现价值对齐并抑制幻觉，支持在线迭代。然而该方法易受奖励黑客影响（策略钻漏洞刷分而非真正提质），且常伴随对齐税（为安全性牺牲下游能力），加之 PPO 训练不稳定及四模型（Actor/Critic/Ref/RM）带来的高昂显存与标注成本，实际应用面临显著挑战。
1.5 对比PPO、GRPO、DPO、DAPO、GSPO
- GRPO：一组prompt经过LLM生成N个结果，每个结果打分，超过平均的被接受，反之被拒绝
- PPO：一组prompt经过LLM生成结果，对结果打分，和上一个的自己比较，有优势被接受，反之被拒绝
- DPO：提前构造好了拒绝和接受的数据，模型参数往接受的方向调整
PPO（需要Critic）
  → GRPO（去掉Critic，组内相对打分）
    → DAPO（修复GRPO在长CoT中的熵崩溃和效率问题）
      → GSPO（token级→sequence级，解决MoE路由不稳定）

DPO（完全离线，绕开RL采样，直接用标注偏好对训练）
2.DPO
2.1 原理
DPO（Direct Preference Optimization）是一种通过人类偏好数据直接优化策略的算法，核心思想在于绕过传统奖励模型训练，简化学习流程。
与传统的强化学习对齐方法对比（例如PPO），PPO的训练至少需要两个阶段：奖励模型训练和策略模型优化，这种方法训练成本高，且需要大量的数据。DPO通过数学变换将两步合并，直接利用偏好数据优化策略，避免显式奖励建模。

2.2实践中怎么选取DPO和PPO
- 任务类型与目标
  - DPO：奖励信号难以定义，但能获取大量偏好排序数据（如人工标注的“好/坏”回答对）
  - PPO：有明确的奖励信号，且需动态探索环境（如自动驾驶中的避障决策）。
- 数据类型：
  - DPO：DPO 依赖离线偏好数据，需要数据覆盖场景全面且标注准确，效果显著（如利用用户评分优化推荐模型）。
  - PPO：PPO 通过在线交互生成数据，适合动态环境（如机器人实时调整抓取策略）。
- 资源与效率要求
  - DPO：DPO 无需训练奖励模型，计算成本低，适合 GPU 资源有限或需快速迭代的场景。
  - PPO：PPO 通过在线探索和奖励模型优化，理论上能达到更高的性能上限，但需更多计算资源。

3.GRPO
3.1 原理
1. GRPO 原理与“消除 Critic”的本质
- GRPO (Group Relative Policy Optimization) 原理：对于同一个输入 Prompt，策略模型（Policy）直接生成一组（Group，比如 $$G=4$$ 或 $8$）不同的回答。对这 $$G$$ 个回答分别计算奖励后，在这一组内部进行比较，得分高于组内平均的给予正向更新，低于平均的给予负向更新。
- 为何能消除 Critic 模型？
  - 在传统 PPO 中，我们需要一个额外的神经网络（Critic/Value Model）来预测“当前状态的预期得分 V(s)”，目的是为了计算优势函数 (Advantage) $$A = R - V(s)$$。Critic 模型通常和 Policy 模型一样大，导致显存（VRAM）和训练显存开销直接翻倍。
  - GRPO 敏锐地发现：既然我们在做大模型生成，完全可以通过对同一个 Prompt 采样多次，用这 $$G$$ 个真实样本的**经验平均值（Empirical Mean）**来替代 Critic 的预测值。
  - 本质：用“多次采样的算力开销”换取了“庞大的 Critic 显存开销”，使得我们在有限的算力下可以训练更大的模型（如 DeepSeek-R1 满血版 671B）。
    
2. “组内平均作基线”的数学推导
面试官想听的不是复杂的积分，而是你对策略梯度定理 (Policy Gradient Theorem) 的理解：
- 数学依据：在策略梯度的基础公式 $$\nabla J(\theta) = \mathbb{E}[ \nabla \log \pi(a|s) (R(s,a) - b(s)) ]$$ 中，数学上已经证明，引入一个只与状态 $$s$$（即 Prompt）有关、与动作 $$a$$（即具体回答）无关的基线 $$b(s)$$，不会改变梯度的期望（即无偏性），但能极大降低梯度的方差。
- 推导过程：
  1. 设对 Prompt $$q$$ 采样了 $$G$$ 个回答 {o_1, o_2, ..., o_G}，得到真实奖励 {r_1, r_2, ..., r_G}。
  2. 我们直接取样本均值 $$\mu = \frac{1}{G}\sum r_i$$ 和标准差 $\sigma$。
  3. 因为 $$\mu$$ 只是针对 Prompt $$q$$ 的固有属性的蒙特卡洛估计，不依赖于某一个具体的 $$o_i$$ 的更新，所以 $$\mu$$ 完美符合基线 $$b(s)$$ 的定义。
  4. 计算优势：   $$A_i = \frac{r_i - \mu}{\sigma + \epsilon}$$
[图片]
- 结论：它把组内表现高于平均的样本 Advantage 算成正数（鼓励），低于平均的算成负数（惩罚）。这就等于用几次生成的算力开销，完美平替了整个 Critic 模型预测 Baseline 的功能，彻底释放了显存！”。
  
3. KL 散度的作用与“特殊设计”
在 GRPO（DeepSeekMath/DeepSeek-R1 论文）中，KL 散度不仅承担了防偏离、防乱码、防熵坍塌的传统作用，它的具体数学设计还做了一个非常精妙的改动：
- 传统的近似做法：通常用 $$\log \pi_\theta - \log \pi_{ref}$$ 来近似 KL。但这种估计在两个分布差异变大时，经常会产生负值或者方差极大，导致优化极不稳定。
- GRPO 的特殊设计（Schulman 估计量）：
GRPO 论文中采用了一种严格正数的无偏估计公式（针对每一个 Token 计算）：
  $$D_{KL} = \frac{\pi_{ref}}{\pi_\theta} - \log \frac{\pi_{ref}}{\pi_\theta} - 1$$
- 面试加分：为什么要这么设计？
  1. 绝对非负性保证：这个公式保证了算出来的 KL 惩罚永远是一个非负数。这解决了传统公式偶尔出现负 KL 导致模型“反而被奖励去偏离 Ref Model”的致命 Bug。
  2. Token 级细粒度惩罚：PPO 往往是在一句话生成完给一个总奖励，而 GRPO 将这个 KL 惩罚直接加在了每一步 Token 的 Advantage 层面。这意味着模型在生成几千个字的推理过程中，每写错一个偏离常理的词，都会被立刻“微小电击”一下，实现了对推理轨迹极其平滑的正则化控制。
因为这种估计没有偏差，且方差较低，能更好的近似KL-散度
总结：
第一，为什么能去掉 Critic。 传统 PPO 的 Critic 跟主模型一样大，显存直接翻倍。GRPO 的思路是对同一个 Prompt 采样八个回答，用这八个回答的平均分当基准线，优势就是得分减均值再除以标准差。这有理论保证——策略梯度里，基准线只要跟具体哪个回答无关，就不会让梯度产生偏差。组内均值完美满足这个条件。本质就是用多采样的算力开销换掉了 Critic 的显存开销。
第二，KL 散度的特殊设计。 常见的 KL 近似方式在两个分布差距拉大时会算出负数，等于反向奖励模型去偏离参考策略，是个很隐蔽的 bug。GRPO 用了一个改进版估计，保证 KL 惩罚永远非负、无偏、方差更低。而且惩罚是落在每个 token 上的，不是生成完整句再算，所以对长链推理的正则化控制非常精细。
第三，为什么 DeepSeek 选这个方案。 GRPO 的代价是采样八次算力翻倍，但 DeepSeek R1 面临的核心问题是模型太大、显存放不下两个。GRPO 正好卡在这个痛点上——不是理论最优，而是工程约束下最合适的选择。

3.2 GPRO的kl散度为啥这么设计
KL-散度的计算公式为：
$$KL(q || p) = \sum_xq(x) log \frac{q(x)}{p(x)} = E_{x \sim q} [log \frac{q(x)}{p(x)}]$$
实际计算中，直接计算KL散度很难，因为需要对所有的x求和或求积分，计算成本和内存消耗过大。因此在实际计算中，往往使用近似方法计算KL-散度。KL-散度有三种估计方式
- K1 估计器：$$K1 = log \frac{q(x)}{p(x)}$$，无偏高方差
- K2 估计器：$$K2 = \frac{1}{2} (log \frac{p(x)}{q(x)})^2$$，有偏低方差
- K3 估计器：$$K3 = \frac{p(x)}{q(x)} - 1 - log \frac{p(x)}{q(x)} $$，无偏低方差
GRPO采用了K3 估计器，因为这种估计没有偏差，且方差较低，能更好的近似KL-散度。


3.3 去掉评估模型，GRPO可能面临的问题
- 奖励估计不准确：Critic Model的核心作用是评估状态或动作的长期价值。在某些任务中，奖励可能只有在完成特定的长期目标后才能获得，这种场景中失去Critic Model会导致奖励估计不准确，从而导致训练不稳定。
以下是DAPO论文中总结的GRPO面临的问题：
- 熵崩溃：策略的熵（不确定性）逐渐趋近于零，导致智能体行为过于确定化，失去探索能力。
- 奖励噪音：奖励信号存在随机波动或误差，导致模型接收到的奖励信息不准确、不稳定，从而影响策略学习的效果。
- 训练不稳定：在强化学习模型的训练过程中，性能指标（如奖励、损失等）出现大幅波动、震荡或不收敛的现象，导致模型无法稳定地学习到有效的策略。

4.dr.GPRO
4.1 为啥要用dr.gpro
在原版 GRPO 里，计算一条回答的总 Loss 时，采用的是‘序列长度归一化’。也就是说，把这句话所有 Token 的 Loss 加起来之后，要除以这条回答的总长度。
这就产生了一个严重的 Bug——模型是非常‘狡猾’的，它发现：如果这道题我做错了，注定要挨罚（也就是 Advantage < 0），那我就疯狂写废话，比如无限循环‘让我再想想’。因为我生成的字数越多，公式里的分母就越大，那么摊到我每个 Token 上的惩罚就被严重稀释了。这就是 Token 聚合偏差，它直接导致了模型为了逃避惩罚而产生毫无意义的‘长篇大论’

第二点，Dr. GRPO 是怎么改的？（全局常数 vs 序列长度）
Dr. GRPO 的改法极其简单且优雅，它直接抛弃了‘序列长度归一化’，改成了**‘全局常数归一化’**。
也就是说，它把公式里的分母，从‘你实际生成的动态长度’，强行换成了一个‘全局固定的常数M
因为 FC 的输出通常很短，我们直接把 M设为 FC 预期的最大 Token 长度（比如 256 或 512）。
第三点，为什么这么改就能抑制长输出？
这就很好理解了。现在分母变成了一个死数字。如果模型再敢为了错题去水 2000 个字的废话，由于惩罚不再被长度稀释，这 2000 个废话 Token 每一个都会吃满实打实的负向梯度惩罚


5.DAPO
5.1 DAPO的核心创新点
- 解耦裁剪：提高clip上界，避免熵崩溃，既保持了思维的严谨性，又让模型的回答充满惊喜创意。
- 动态采样：过滤掉准确率为 1 和 0 的数据，自动过滤掉太简单或超纲的题目，提升训练效率和稳定性。
- token级梯度损失：提升长序列样本中的token对整体损失的影响，使得模型能够更好地学习长序列中的推理模式（long-CoT）。
- 过长样本奖励调整：对过长样本的惩罚进行平滑处理，用渐进式调整让模型明白：不是文章越长越好，而是要把复杂问题说清楚。
- 移除了 KL 散度惩罚（KL Divergence Penalty）。但在 DAPO（或者说复现 DeepSeek-R1）的纯推理场景下，我们的目标本来就是要让模型打破旧的思维定势，发展出长链条推理、自我纠错等初始模型根本没有的涌现能力。在 System 2（慢思考）的强化学习中，我们要敢于拿掉 KL 散度的‘紧箍咒’，转而用 Token-level Loss 治废话，用 Clip-Higher 拓宽探索，用 Dynamic Sampling 榨干算力。 

5.2 为什么小模型微调FC不用DAPO 去用dr.gpro?
1. 参数容量与 Clip-Higher 陷阱：DAPO 放开截断上界（Clip-Higher），其底层假设是大模型有足够的参数量去产生‘多样化的高质量探索（顿悟）’。但小模型容量极其有限，它探索不出新花样，放开上界反而会引入严重的梯度不稳定，极易导致训练崩溃。
2. 短输出场景的梯度劣化：DAPO 的 Token-level loss 是为了逐字惩罚‘万字长考’里的废话。但在 FC 这种极短输出（仅几百 Token 的 JSON）场景下，Token-level 计算带来的细粒度方差过大，提供的梯度信号质量反而更差。Dr. GRPO 的全局常数归一化在短序列下梯度要平稳得多。
3. 防灾难性遗忘与零容错：DAPO 激进地去掉了 KL 散度，小模型底子薄，一旦失去 KL 保护去‘自由探索’，极易丧失正常对话能力变成乱码机器。而 Dr. GRPO 既保留了 KL 散度保底，又用无情的全局常数归一化，让多说的任何一句废话都吃满全额负惩罚。
总结：对小模型做 FC，绝不能用 DAPO 搞‘开放式顿悟’。Dr. GRPO 用**‘稳梯度、留 KL、严惩罚’**的硬核手段，能以极低的试错成本，硬生生把‘精准输出 JSON 且立刻闭嘴’刻进小模型的肌肉记忆里。”

6.GSPO
GSPO 是阿里 Qwen 团队在训练 Qwen3（含 Qwen2.5-VL 等）时提出的一种具有“釜底抽薪”意义的新算法。 如果说 DAPO 和 Dr. GRPO 还是在 GRPO 的框架里缝缝补补，那么 GSPO 直接推翻了从 PPO 时代遗留下来的“祖宗之法”。

---
一、 核心痛点：GRPO 到底遇到了什么不可原谅的 Bug？
随着大模型参数量飙升，特别是进入 长链条推理（System 2） 和 MoE（混合专家架构） 时代，原版 GRPO 暴露出了两个致命问题：
1. “奖励”与“优化”的粒度严重错位（高方差灾难）
  - 痛点：在 GRPO/PPO 中，计算新旧模型差距（Importance Ratio，即重要性采样权重）是逐个 Token (Token-level) 计算的。但是，系统给的奖励（比如代码跑通了没有）明明是针对整句话 (Sequence-level) 给的！
  - 后果：拿着全局的奖励去强行分配给每一个 Token 算概率比，会引入极其巨大的高方差噪声。在生成几千个 Token 的长推理时，这种噪声会疯狂累加，导致梯度上蹿下跳，训练极易崩溃。
2. MoE 模型的“专家崩塌 (Expert Collapse)”
  - 痛点：因为 GRPO 的 Token 级梯度极不稳定，在训练稀疏激活的 MoE 模型时，极容易导致某几个专家（Experts）的权重被瞬间“打飞”，从而发生模型崩溃。以前为了稳住 MoE 的 RL 训练，必须要用 Routing Replay（路由回放） 这种极其恶心且吃显存的工程补丁。

---
二、 GSPO 的颠覆性解法：从 Token 级到 Sequence（序列）级
GSPO 的核心思想就一句话：既然奖励是按整句话（Sequence）给的，那重要性采样和截断（Clipping）也必须按整句话来算！
- 原版 GRPO 的做法：每个 Token 都有自己的截断（Clip）和权重。一个长句子里，有的字被截断了，有的字没被截断，梯度被割裂得七零八落。
- GSPO 的做法：
  1. 计算当前生成的整段话在新模型和旧模型下的联合概率比（然后除以长度取平均，防止数值爆炸）。
  2. 对这个序列级的概率比进行 Clip 截断。
  3. 众生平等：一旦算出了这句话的总权重，这句话里的所有 Token 共享这同一个权重梯度。
- 面试人话解释：“GSPO 不再盯着单个字的得失，而是把一整条推理轨迹当成一个不可分割的整体（Sequence-level）来评估。整条轨迹表现稳，里面的所有字就一起稳步更新；整条轨迹偏离太远，就整个一起踩刹车。”

---
三、 GSPO 带来的三大“降维打击”级收益（怎么用？）
在面试中，你一定要强调 GSPO 不仅仅是数学公式的变动，更是工程架构上的巨大胜利：
3. 极致稳定，彻底拯救 MoE 训练
- 收益：因为序列级优化极大地抹平了 Token 级的抖动噪声，GSPO 实现了前所未有的训练稳定性。它彻底扔掉了 Routing Replay 等复杂的补丁机制，直接让超大规模的 MoE 模型平稳收敛，永不崩塌。
4. 完美的 Scaling Law（告别算力空转）
- 收益：以前用 GRPO 训练，步数稍微长一点模型就“学傻了”（Collapse）。而 GSPO 被证明具有极强的可扩展性：你投入的算力越多、生成的序列越长、训练的时间越久，模型的性能就会持续稳步上升。这是 Qwen3 能在推理榜单上霸榜的核心底气。
5. 对 RL 训练架构极其友好（Infrastructure-Friendly）
- 收益（极其硬核的加分项）：由于 GSPO 只需要“整句话”的概率，它对精度的容忍度极高。这意味着，我们可以直接使用推理引擎（如 vLLM, TensorRT-LLM）返回的粗粒度概率做优化，而不需要再把数据送回庞大的训练引擎里逐字重算一次 Logprobs！这为当下最前沿的**“训推分离架构（Training-Inference Disaggregation）”**扫清了最大的障碍。

7.奖励
7.1 大模型语义评估（COR）
核心思想：先定标准，再打分
- 普通奖励模型：看一遍回答，直接输出一个分数（如 0.8）。为什么是这个分数？不知道。
- CoR 奖励模型：看回答前，先根据问题类型生成一份评分标准（Rubrics），然后对照标准一步步分析，最后输出带有推理链的判断。
先写评价维度（评四个维度：函数名对不对、参数有没有幻觉、参数值业务逻辑合不合理、多轮上下文是否一致），再分析，最后给分。”
这就是 CoR 的具体实践。这四件事就是你的 Rubrics（评分标准）。Judge 模型需要：
1. 生成 Rubrics：明确要从哪几个维度打分（函数名、参数幻觉、业务逻辑、上下文一致性）。
2. 逐项分析：对照这两个候选输出（正确 vs 错误），在每个维度上给出判断。
3. 输出结论：基于各维度的分析，给出最终偏好判断。
2. 为什么叫“链”（Chain）？
因为它让推理过程显式化、可追溯。你可以在训练日志里看到：
- “第一步：我检查了函数名 → 候选 A 正确，候选 B 错误。”
- “第二步：我检查了参数值 → 候选 A 符合业务规则，候选 B 数量不是 100 的倍数。”
- “结论：候选 A 更优，+1 分奖励给 A。”
这种“链式”推理，就是 CoR 的核心思想。
4. 特别说明：有两个不同的“CoR”
这里需要区分一下，学术界有两个常见的 CoR 缩写，不要混淆：
暂时无法在飞书文档外展示此内容
CoR（Chain of Reasoning）的三步走——写评价维度 → 分析 → 打分——核心目的是防止 LLM-as-Judge 的两个常见偏差。第一是 Position Bias：如果直接让 Judge 给分，它容易因为输出顺序（先看到的内容打分更高）产生偏差，强制先写评价维度让它在比较前明确知道自己要看什么，减少顺序干扰。第二是"分数先行"偏差：如果让 Judge 直接输出分数，它会基于直觉先定分，然后找理由，推理链是倒置的，鲁棒性差。CoR 格式强制推理在先、打分在后，推理链可以被我们审计，如果发现 Judge 在某类 case 上推理链有规律性错误（比如误判了多条件 AND 查询），可以针对性地优化 Judge Prompt。
