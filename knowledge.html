Transformer & 注意力机制

从笔试真题出发，覆盖 Self-Attention、MHA、GQA、MLA、Flash-Attention、稀疏注意力等全体系，横向扩展所有关联变体。

核心机制
⚡
Scaled Dot-Product Attention（缩放点积注意力）
真题考点：为什么要除以 √d_k ？GQA 如何分组？
▶
核心考点 标准公式
Attention(Q,K,V) = softmax( QKᵀ / √d_k ) · V

除以 √d_k 的原因：当 d_k 较大时，QKᵀ 的点积方差为 d_k，数值会很大，softmax 会进入梯度极小的饱和区（近似 one-hot），梯度消失。除以 √d_k 将方差归一化为 1。

横向扩展 不同注意力变体对比
变体公式/机制关键特点
Dot-ProductQKᵀ · V未缩放，d_k大时梯度消失
Scaled Dot-ProductQKᵀ/√d_k · V标准Transformer使用
Additive (Bahdanau)tanh(W₁Q + W₂K)早期seq2seq，参数量大
Linear Attentionφ(Q)(φ(K)ᵀV)复杂度降至O(n)，近似
Sparse Attention仅计算部分注意力Longformer/BigBird
横向扩展 Multi-Head Attention (MHA)
MHA(Q,K,V) = Concat(head₁,...,headₕ) · W^O headᵢ = Attention(QWᵢQ, KWᵢK, VWᵢV)

多头注意力的输出方式：各头结果拼接后线性投影（不是取均值，不是门控）。

参数量：每个头 d_k = d_v = d_model / h，总参数量固定，与头数无关（因为头越多，每头维度越小）。

横向扩展 GQA / MQA / MLA 对比

🧠 记忆口诀

GQA：h头分g组，每组 h/g 个头共享同一份K/V。题目问"h=8,g=2，每组几个头共享"→ 8/2 = 4个头。

横向扩展 Flash-Attention

核心思想：分块（Tiling）计算，避免将完整 N×N 注意力矩阵写入 HBM（高带宽内存），将显存复杂度从 O(N²) 降至 O(N)。

序列并行度提升：Flash-Attention v2 将序列分块到多个设备，每设备处理的序列更短，但总显存几乎不变（各设备单独计算，总量不减）。

⚠️ 易错点

Flash-Attention 减少的是显存访问次数（IO bound），不是总显存需求；序列并行不改变总显存。

位置编码（Positional Encoding）
真题：Positional Encoding 的作用是什么？
▶
核心考点 为什么需要位置编码

Self-Attention 本身对位置不敏感（排列等变），同一句话词序改变后 attention 输出不变，所以需要显式注入位置信息。

⚠️ 高频错误

Positional Encoding 的目的不是解决 RNN 梯度消失问题——Transformer 完全抛弃了 RNN。

横向扩展 降低复杂度的方法汇总

LayerNorm vs RMSNorm vs BatchNorm
真题：RMSNorm 取代 LayerNorm 的直接收益？Dropout 推理时关闭？
▶
核心考点 归一化方法对比
方法归一化维度均值计算适用场景
BatchNormbatch维度✅CNN，batch > 1
LayerNorm特征维度✅Transformer
RMSNorm特征维度❌（省去）LLaMA等新LLM
InstanceNorm单样本特征✅风格迁移
GroupNorm分组特征✅小batch CNN

RMSNorm 优势：去掉均值计算（中心化），推理与训练均省去一次减法，计算更快；LLaMA、Mistral 等均使用。

核心考点 Dropout 推理时行为

Dropout 在推理时关闭（不随机丢弃），通常在训练时用 Inverted Dropout（权重乘以 1/p 保持期望不变），因此推理时直接使用全部权重，推理时间不变。

⚠️ 典型陷阱

加了 Dropout 的网络推理时间不会增加（Dropout 关闭），和没加 Dropout 时推理时间相同。

RLHF 完整流程
真题：reward model 训练依赖什么？RLHF 典型算法是？
▶
核心考点 RLHF 三阶段
  1. SFT（监督微调）：在高质量指令数据上做监督学习，使模型学会遵循指令
  2. Reward Model 训练：基于人类偏好排序数据训练奖励模型（不是监管数据，不是随机初始化）

LoRA 参数高效微调
真题：LoRA 的优点？是否降低推理显存？
▶
核心考点 LoRA 原理
W' = W₀ + ΔW = W₀ + BA B ∈ R^(d×r), A ∈ R^(r×k), r << min(d,k)

原模型权重 W₀ 冻结，只训练低秩矩阵 B 和 A，参数量大幅减少（约原模型的 0.1%~1%）。

核心考点 LoRA 优缺点
维度结论
参数量✅ 大幅减少（只训练 BA）
原模型权重✅ 保持不变（冻结）
任务切换✅ 不同任务用不同 LoRA 模块，快速切换
推理显存⚠️ 不一定降低（合并后不变，不合并略增）
推理速度⚠️ 不一定更快（合并后相同，不合并略慢）
⚠️ 常见误区

LoRA 降低的是微调时的参数量和显存，推理时需要合并 ΔW 到 W₀，合并后显存和标准模型一样。

⚠️ 温度参数陷阱

低温度 → 输出更集中（保守，多样性低）；高温度 → 输出更分散（多样，随机）。要增加多样性应提高温度，而非降低。

横向扩展 推理阶段优化方法

  • 量化（INT8/INT4）：减少模型权重精度，降低显存和推理延迟

  • DPM-Solver：扩散模型专用高效 ODE 求解器，大幅降低采样步数
  • DDIM：非马尔可夫采样，20步达到DDPM 1000步效果

涌现能力 & 大模型特性
真题：涌现能力的定义？预训练是否需要标注数据？
▶
核心考点 涌现能力（Emergent Abilities）

定义：当模型规模（参数量、训练数据量）达到某个临界阈值时，突然展现而小模型不具备或表现极差的能力。

典型例子：GPT-3 的 few-shot 学习、链式推理（Chain-of-Thought）、指令遵循。

涌现能力是规模扩大后突然出现（不是缩小，不是手动添加），与硬件无关。

🧠 ReLU 考点

输入 [0,1,2,3] → ReLU → [0,1,2,3]（全非负，原样输出）。输入 [-1,0,1] → ReLU → [0,0,1]。

核心考点 Softmax 详解
softmax(xᵢ) = exp(xᵢ) / Σⱼ exp(xⱼ) 数值稳定版本：xᵢ -= max(x) # 先减最大值，数学等价

Softmax 作用：将任意实数向量映射到[0,1] 区间且和为1的概率分布（不是[-1,1]，不是对数概率，不是防梯度爆炸）。

与交叉熵结合：softmax 输出概率 → 交叉熵计算 log loss，反向传播梯度简洁。

Hessian-Vector Product & 二阶优化
真题：Hv 的核心计算思路？Fisher 信息近似？
▶
核心考点 高效计算 Hv

直接计算 Hessian 矩阵的复杂度为 O(n²)，但 Hessian-Vector Product 可以用反向传播高效计算：

  1. 正常计算梯度 g = ∇L(θ)
  2. 将 g 视为新的"损失"（标量 g·v），对 θ 再次做反向传播
  3. 得到的梯度即为 Hv
Hv = ∇(gᵀv) # 二阶反向传播，复杂度 O(n)

朴素贝叶斯 & 贝叶斯定理
真题：高斯NB适合连续还是离散？先验/后验概率区别？
▶
核心考点 贝叶斯公式
P(c|x) = P(x|c) · P(c) / P(x) 后验 ∝ 似然 × 先验
  • 先验（Prior）P(c)：观测数据前的初始概率估计
  • 似然（Likelihood）P(x|c)：已知类别时，观测到特征的概率
  • 后验（Posterior）P(c|x)：观测到数据后更新的概率，不超过1
核心考点 三种朴素贝叶斯
类型适合特征P(x|c) 假设典型应用
高斯 NB连续型（实数）正态分布鸢尾花分类
多项式 NB离散（计数/频次）多项式分布文本分类
伯努利 NB二值特征（0/1）伯努利分布垃圾邮件
⚠️ 高频陷阱

高斯朴素贝叶斯适合连续型特征（假设正态分布），不适合离散型特征。

横向扩展 垃圾邮件分类流程

P(垃圾|关键词) ∝ P(关键词|垃圾) · P(垃圾) → 用贝叶斯定理根据关键词更新邮件是垃圾的后验概率。

朴素假设：各关键词在给定类别下相互独立 → P(x|c) = Π P(xᵢ|c)。

拉普拉斯平滑：P(w|c) = (count(w,c)+1) / (Σ count(w',c) + V)，防止零概率。

方法分量使用可并行？收敛速度
Jacobi使用上一轮所有分量✅ 可并行较慢

横向扩展 正交矩阵性质

  • Aᵀ = A⁻¹（转置等于逆矩阵）
  • det(A) = ±1
  • 列向量两两正交且单位长度，一定线性无关
  • 保持向量长度（等距变换）：‖Ax‖ = ‖x‖
# 考题：3×3正交矩阵，第一行(a,a,a)三个相等正数 # AᵀA=I → 每行模为1 → 3a²=1 → a=1/√3 = √3/3
📐
矩阵的秩
真题：rank(AB)的上界？初等行变换改变秩吗？
▶
核心考点 秩的重要性质
  • 行秩 = 列秩 = 矩阵的秩（两者始终相等）
  • 初等行变换不改变矩阵的秩（行化简求秩的理论基础）
  • rank(AB) ≤ min(rank(A), rank(B))
  • 若A或B为零矩阵，则 rank(AB) = 0
  • A可逆时，rank(A) = rank(A⁻¹) = n
⚠️ 常见错误

rank(AB) ≥ rank(A) 是错误的（正确是 ≤）。矩阵的秩不等于行空间维数（行秩=列秩=秩，与哪个维数都相等）。

病态问题 & 差分近似
真题：病态问题的表现？中心差分的误差阶？
▶
核心考点 病态问题

病态（ill-conditioned）：输入的微小扰动导致输出显著变化。典型表现：线性方程组的系数矩阵行列式接近0（条件数大，矩阵接近奇异）。

⚠️ 不是病态的情况

数值积分步长大（截断误差，不是病态）；牛顿迭代初值不当（收敛问题，不是病态）；浮点舍入误差（精度问题，不是病态）。

核心考点 差分近似误差
# 前向差分（一阶精度 O(h)） f'(x) ≈ [f(x+h) - f(x)] / h # 后向差分（一阶精度 O(h)） f'(x) ≈ [f(x) - f(x-h)] / h # 中心差分（二阶精度 O(h²)）← 真题 f'(x) ≈ [f(x+h) - f(x-h)] / (2h) # 二阶项在 Taylor 展开中相消，误差从h²开始
横向扩展 有效数字 & 消去误差
  • 有效数字：从第一个非零数字到最后一位可靠数字
  • 消去误差（Catastrophic Cancellation）：两个相近数相减，有效数字位数大幅降低
  • 例：2.0050×10⁶ - 1.9950×10⁶ = 0.0100×10⁶，只剩2-3位有效数字
  • min-max 归一化对量级相近的数据会导致有效数字减少

模型量化 & 三次样条
真题：INT8量化流程？三次样条的连续性条件？
▶
核心考点 对称 INT8 量化
s = max(|x|) / 127 # 缩放因子 Q = clip(round(x/s), -127, 127) # 量化 x_deq = Q × s # 反量化 # per-sample（按行）: 对矩阵A每行单独计算s # per-channel（按列）: 对矩阵B每列单独计算s # 整数域矩阵乘法后反量化：Y = Q_A @ Q_B，再 × (s_A · s_B)
核心考点 三次样条插值
  • 在每个子区间用三次多项式近似
  • 在节点处保证函数值、一阶导数和二阶导数都连续（C²连续）
  • 插值函数通过所有给定数据点
  • 相比高次多项式插值，能避免 Runge 现象（端点振荡）
⚠️ 错误说法

"三次样条在节点处二阶导数不连续"——这是错误的，三次样条的核心优势正是保证了二阶导数的连续性（C²）。
